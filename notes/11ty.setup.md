### Q: How can I use 11ty to blog this project from docs/ via GitHub Pages?
**Date:** 2026-02-12

## 1. Initialize and install 11ty

```bash
npm init -y
npm install @11ty/eleventy
```

## 2. Create `.eleventy.js` config

This tells 11ty to read from `docs/` and output to `_site/`:

```js
module.exports = function(eleventyConfig) {
  // Copy static assets if you add any later
  // eleventyConfig.addPassthroughCopy("docs/assets");

  return {
    dir: {
      input: "docs",
      output: "_site",
      includes: "_includes",  // docs/_includes for layouts
      data: "_data"           // docs/_data for global data
    },
    markdownTemplateEngine: "njk"
  };
};
```

## 3. Create a layout

Create `docs/_includes/base.njk`:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>{{ title | default("metagen") }}</title>
  <style>
    body { font-family: system-ui, sans-serif; max-width: 48rem; margin: 2rem auto; padding: 0 1rem; line-height: 1.6; }
    pre { background: #f4f4f4; padding: 1rem; overflow-x: auto; }
    table { border-collapse: collapse; width: 100%; }
    th, td { border: 1px solid #ddd; padding: 0.5rem; text-align: left; }
  </style>
</head>
<body>
  <nav><a href="/">Home</a> | <a href="/reports/">Reports</a> | <a href="/qa/">Q&A Log</a></nav>
  <main>{{ content | safe }}</main>
</body>
</html>
```

## 4. Add front matter to your markdown files

Add YAML front matter to the top of each markdown file so 11ty knows what layout to use. For example, create `docs/index.md` as your homepage:

```md
---
layout: base.njk
title: metagen — Project Blog
---

# metagen

Welcome to the metagen project blog. This tool automatically generates
standards-compliant metadata for geospatial and tabular datasets.

## Recent Reports

{% for report in collections.reports %}
- [{{ report.data.title }}]({{ report.url }})
{% endfor %}
```

Add front matter to existing files like `docs/reports/gap_report_2026-02-12_211401.md`:

```md
---
layout: base.njk
title: Gap Report 2026-02-12
tags: reports
---
```

And to `docs/claude/qa-log.md`:

```md
---
layout: base.njk
title: Q&A Log
permalink: /qa/
---
```

## 5. Add npm scripts to `package.json`

```json
"scripts": {
  "build": "eleventy",
  "serve": "eleventy --serve"
}
```

## 6. Deploy with GitHub Actions

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: _site

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

## 7. Enable GitHub Pages

In your repo settings on GitHub:
1. Go to **Settings > Pages**
2. Under **Source**, select **GitHub Actions**

## 8. Update `.gitignore`

Add these:

```
_site/
node_modules/
```

## Summary

| Piece | Path |
|---|---|
| 11ty config | `.eleventy.js` |
| Content source | `docs/` |
| Layouts | `docs/_includes/` |
| Build output | `_site/` (gitignored) |
| Deploy | GitHub Actions → GitHub Pages |

Run `npm run serve` locally to preview at `http://localhost:8080`. Every push to `main` will auto-deploy to `https://acatejr.github.io/metagen/`.
