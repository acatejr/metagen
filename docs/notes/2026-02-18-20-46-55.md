### Q: How to leverage Hugo tags in the metagen documentation site
**Date:** 2026-02-18

Tags are a built-in Hugo taxonomy. Add `tags` to any page's front matter — Hugo auto-generates `/tags/` (index) and `/tags/<tagname>/` (per-tag listing) pages. No `hugo.toml` changes needed.

**Recommended tag taxonomy for metagen:**

| Category | Tags |
|---|---|
| Metadata standard | `iso-19115`, `fgdc`, `dcat`, `dcat-us` |
| Source format | `wsdl`, `shapefile`, `csv`, `geojson`, `postgis`, `oracle` |
| Content type | `gap-report`, `crosswalk`, `qa`, `note` |
| Topic | `ai-enrichment`, `llm`, `spatial`, `tabular`, `publisher` |

**Example front matter:**
```yaml
tags:
  - fgdc
  - dcat-us
  - gap-report
  - wsdl
```

**Caveat:** The `hugo-book` theme sidebar is driven by sections, not tags. Tags appear at the bottom of each page and at `/tags/` — useful for cross-cutting discovery but not sidebar navigation.
